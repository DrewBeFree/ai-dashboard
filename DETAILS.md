# AI Dashboard — Technical Details

## Token Counting Notes

Claude Code tracks four token categories per model:

| Field | Description |
|---|---|
| `inputTokens` | Tokens in the prompt (not from cache) |
| `outputTokens` | Tokens generated by the model |
| `cacheReadInputTokens` | Prompt tokens served from the prompt cache (cheaper) |
| `cacheCreationInputTokens` | Tokens written to the prompt cache |

The `outputTokens` count dominates the API-equivalent cost for heavy coding workloads since output is priced 5× higher than input. Cache reads are cheap (~10% of input cost) but accumulate to large numbers over long sessions.

## Why `costUSD` is Always 0

Claude Code sets `costUSD: 0` for all models when you're on a Max/Pro subscription. The cost is paid as a flat monthly rate, not tracked per-token. The dashboard recalculates an API-equivalent value using current published pricing to show what the usage would cost at pay-per-token rates.

## Ollama API

The dashboard queries `GET /api/tags` (unauthenticated). Response structure:
```json
{
  "models": [
    {
      "name": "llama3.2:latest",
      "model": "llama3.2:latest",
      "size": 2019393189,
      "details": { "family": "llama", "parameter_size": "3.2B" }
    }
  ]
}
```

## OpenAI Usage API

Endpoint: `GET https://api.openai.com/v1/usage?date=YYYY-MM-DD`

Returns token counts aggregated by model for the given date. Requires an API key with `usage.read` permission. Note: the usage API returns today's data only per call — building a 30-day view requires 30 separate requests.

## Port

The server binds to `127.0.0.1:7474` (localhost only). Not accessible from other machines on the network.

## Claude Stats File Location

`~/.claude/stats-cache.json` — written and updated by the Claude Code daemon. The `lastComputedDate` field shows when it was last refreshed. The file is not updated in real-time during a session; Claude Code recomputes it periodically.

## Chart.js Version

Using Chart.js 4.4.0 from jsDelivr CDN. Requires internet access on first load (cached by browser afterward).
